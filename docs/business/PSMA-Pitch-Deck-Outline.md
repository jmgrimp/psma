# PSMA — Pitch Deck Outline (Optional)

## 1. Title
- Program Subscription Manager Application (PSMA)
- “Subscribe to shows, not services.”

## 2. The problem
- Streaming fragmentation
- Manual work to track where/when
- Subscription creep and wasted spend

## 3. The solution
- Aggregate availability (US)
- Users pick shows + preferences
- PSMA generates a subscription plan
- Reminders and change awareness

## 4. Why now
- More exclusive content
- More services
- More churn and user frustration

## 5. Differentiation
- Planning + timing optimization
- Preference-driven recommendations
- Proactive changes + confirmation flow

## 6. AI role
- Explains changes and proposes updates
- Role-based agent profiles (scheduler/researcher)
- Confirmation-first and transparent

## 7. MVP approach
- Local-first, low cost
- Best-effort data coverage
- Provider plugin model

## 8. Roadmap
- Phase 1: internal MVP
- Phase 2: provider expansion + better fidelity
- Phase 3+: optional automation + additional UIs

## 9. Risks & mitigations
- Data coverage/quality
- Terms-of-service
- Trust and explainability

## 10. Ask / next steps
- Align on MVP scope and internal pilot success metrics
- Begin build after architecture RFC and provider selection
